## 💡 주요 개념 요약

### 동시성 문제의 핵심
- **공유 자원**: 여러 스레드가 동시에 접근하는 데이터
- **Race Condition**: 실행 순서에 따라 결과가 달라지는 현상
- **원자성**: 연산이 중간에 끊기지 않음
- **가시성**: 변경사항이 다른 스레드에게 보임

### 해결 방법
1. **메모리 레벨**: synchronized, Lock, volatile
2. **DB 레벨**: 비관적 락, 낙관적 락
3. **선택 기준**: 충돌 빈도, 성능 요구사항, 비즈니스 로직 복잡도

### 실무 가이드
- 공유 자원 명확히 파악
- 최소 범위만 동기화
- 적절한 락 선택
- 반드시 테스트 코드 작성

## 🤝 Q&A 예상 질문

### 기본 질문
1. **Q**: 스레드와 프로세스의 가장 큰 차이는?
   **A**: 메모리 공유 여부. 스레드는 Heap을 공유, 프로세스는 독립적.

2. **Q**: 왜 동시성 문제가 발생하나요?
   **A**: 여러 스레드가 공유 자원을 동시에 수정하려고 하기 때문.

### 심화 질문
3. **Q**: 비관적 락과 낙관적 락, 언제 뭘 써야 하나요?
   **A**: 충돌이 자주 발생 → 비관적 락, 충돌이 드묾 → 낙관적 락

4. **Q**: 분산 환경에서는 어떻게 동시성을 제어하나요?
   **A**: Redis Lock, Redisson 같은 분산 락 사용

5. **Q**: Virtual Thread는 일반 스레드와 어떻게 다른가요?
   **A**: OS 스레드와 1:1 매핑이 아닌 경량 스레드, Java 21+
